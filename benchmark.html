Traditional CPU architectures process neurons sequentially — one at a time, every tick. At 250K neurons that means 250,000 operations per sweep. The VANA engine's route-based sparse routing evaluates route conditions each tick but only fires routes whose conditions are met, skipping all inactive neurons entirely. This is real measured timing from the Sanguis V3.0 engine running on an i7-12700K.

* Architectural Note: All V2 Sparse Skip timing data is measured from actual Sanguis V3.0 engine runs using real .sang payload files on an Intel i7-12700K CPU with 32GB RAM. The VANA engine's route-based sparse routing skips inactive neurons each tick — only routes whose conditions are met execute. Sequential CPU times are computed as neurons × 1µs (theoretical O(N) baseline). V2 Dense Route times are sparse_time × route_count (all routes firing). V2 Region Skip is a partial optimization estimate at 65% of dense. Payloads are available as .sang source files in the Sanguis distribution.
