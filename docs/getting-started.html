Getting Started

What Sanguis is, how to install it, and your first organism

What Sanguis Is

Sanguis is a genome language. You do not write instructions. You describe a living circuit — what it holds, when it fires, and how it wires to other circuits. The VANA engine reads that description, boots the organism, and the route starts ticking.

You do not program in Sanguis. You grow organisms.

A .sang file is an organism's genome. When VANA reads it, the organism boots, the route starts ticking, and it lives.

The Pipeline

.sang  →  [parse]  →  Genome (AST)  →  [VANA engine]  →  life

.sang — Sanguis source (the genome, human-written)
.exe — Native Windows PE binary (via sanguis native)

No intermediate representation. The engine boots directly from the parsed AST. The flow command and .sirc format were removed in V3.4.

Installation

Prerequisites

Windows 10 or later
GPU optional (CUDA for cortex-scale, not required for basic organisms)

Option A — SDK (No Build Required)

When the public repository is released, it will include a pre-built SDK with all required binaries. You will not need Cargo, a Rust toolchain, or any build step.

1. Download the SDK folder from the repository
2. Add sanguis.exe to your PATH
3. Optionally add vana_bench.exe for CUDA benchmarking

The SDK includes the pre-built CLI binary, CUDA benchmark tool, example organisms, and documentation. The full source (Rust crates, CUDA source, build scripts) is included in the repository for those who want to build from source or contribute.

Option B — Build From Source

sanguis drip

This runs cargo build --release and copies the binary to ~/bin/sanguis.exe. The sanguis command is now available system-wide.

Verify

sanguis version
# Output: sanguis IT3 V3.4 — programs are alive

Your First Organism

Create a file called hello.sang:

body Hello {
  hold greeting : text = "Hello, world!"
  when start {
    emit greeting to screen
  }
}

Compile and run:

sanguis run hello.sang
# Output: Hello, world!

That's it. You grew an organism. It has one organ (Hello), one hold (greeting), one route (start), and one emit (print the greeting to screen). The route ticked once, the organism spoke, and you saw it.

A more complete organism

// minimal.sang — the smallest living organism
//
// Three organs: Body senses, Brain thinks, Soul remembers.

circuit Body {
    hold sensors : live   field [16] = 0.0    // 16 sensory neurons
    hold motors  : live   field [4]  = 0.0    // 4 motor neurons

    when sense {
        emit read_sensors() to sensors
    }

    when move {
        but motors is not empty
        emit actuate(motors) to world
    }
}

circuit Brain {
    hold s1      : live    field [16] = 0.0
    hold l2      : learned field [16] [16] = 0.0
    hold c4      : aware   field [16]
    hold c4_conf : aware   scalar

    when think {
        but Body.sensors is not empty
        emit Body.sensors to s1
        emit s1 to motors
    }
}

circuit Soul {
    hold states : memory field [16] [64]
    hold count  : learned scalar = 0

    when save {
        but Brain.c4_conf < 0.8
        emit write(Brain.s1, Brain.c4) to disk "vana_home/e3_store"
        emit count += 1 to screen
    }

    when boot {
        but disk has e3
        emit read(all) to states
    }
}

// Connectome — how the organs wire together
wire Body.when_sense to Brain.when_think
wire Brain.when_think to Soul.when_save
wire Brain.emit to Body.when_move

45 lines. It senses, thinks, remembers, and acts. The route breathes through it.

The CLI

sanguis native <file.sang> [-o <out>]   Compile .sang to native .exe (Windows PE)
sanguis run <file.sang> [ticks]         Boot organism, run the route
sanguis run <file.sang> --stdin         Read sensor data from stdin (one line per tick)
sanguis run <file.sang> --quiet         Suppress boot messages
sanguis parse <file.sang>               Parse .sang, print AST summary
sanguis check <file.sang>               Check for errors (parse without running)
sanguis drip                             Rebuild the Rust bootstrapper
sanguis version                          Print version

Examples

# Run an organism for 1000 ticks
sanguis run genome/minimal.sang 1000

# Feed sensor data from stdin (space or comma separated floats)
echo "0.5 0.3 0.1 0.8" | sanguis run genome/minimal.sang 1 --stdin

# Compile to a standalone .exe
sanguis native hello.sang
# Produces hello.exe — no Rust runtime, no dependencies

# Check syntax without running
sanguis check genome/minimal.sang
