Native Compilation & File Formats

.sang to .exe — the self-hosting compiler pipeline and file formats

Pipeline

.sang source → [phase1.sang] → .vana genome → [phase2.sang] → patch.ps1 → [PE assembler] → .exe

The .exe is fully standalone — no runtime dependencies, no external compiler, no assembler, no linker.

Running the Compiler

vana_runner.exe phase1.sang    # produces output.vana
vana_runner.exe phase2.sang    # produces patch.ps1
powershell patch.ps1           # produces output.exe
output.exe                     # runs the compiled program

Native Output Builtins

V4 provides two builtins for native output generation:

BuiltinSignatureReturnsDescription
write_byteswrite_bytes(data)scalarWrite raw bytes to native output
write_tcodewrite_tcode(hex)scalarWrite a tcode (hex-encoded opcode) to native output

tcodes

Tcodes are x86-64 opcodes encoded as Sanguis tokens. The PE template currently supports:

Tcodex86-64 Operation
sub_rspAllocate stack space
get_stdoutGet stdout handle
save_stdoutStore stdout handle
load_scalarLoad a scalar value
store_scalarStore a scalar value
check_condConditional jump (jle/jne/jge)
write_stringWrite text to stdout
exitExitProcess

File Formats

Every file type in the Sanguis V4 ecosystem

.sang — Sanguis Source

Human-written text file containing circuit declarations, holds, routes, conditions, and emits. Comments with //. This is the primary source format.

.vana — Genome Format

Pipe-delimited text format produced by phase 1 of the compiler. Each line is a token:

VANA|v1
BODY|name
HOLD|name|memory scalar|0.0
WHEN|trigger
BUT|condition
EMIT|value|target
THROUGH|holdname
STREAM|holdname
ROUTE|src|tgt
ENDBODY

.exe — Native PE Binary

Standalone Windows x86-64 executable. No runtime dependencies. Generated by the PE assembler from tcodes.
