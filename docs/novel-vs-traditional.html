What's Novel vs Traditional Programming

For programmers coming from C, Python, Rust, and everything else

This section is for programmers coming from traditional languages (C, Python, Rust, etc.). Here's what Sanguis does differently.

No control flow statements

TraditionalSanguis
if (x > 5) { ... }but x > 5.0 (gates a route)
for (i=0; i<N; i++)repeat ... until cond (V3)
while (cond)The route ticks forever
return valueemit value to target
function callCircuit fires via wire

There is no if. There is no for (in V2). There is no while. There is no return. The route is the only control flow. Seven stages, four layers, one breath. Every circuit ticks on every route pass. You cannot skip stages. You cannot reorder them. The route is the heartbeat.

Layers are types

Traditional typesSanguis layers
int, floatlive scalar, learned scalar
stringmemory text
float[]live field [N]
float[][]learned field [N] [M]
bool(no bool — use scalar > 0.5)

The lifetime is the type. A live value dies next tick. A learned value survives the session. A memory value survives death (power off). An aware value is computed fresh every breath. You cannot assign a memory to a live without the route carrying it. The layers are not interchangeable. Data must travel the route to change layers.

No assignment — data flows

// Traditional: x = 5
// Sanguis:    5 to x          (inside an emit)

// Traditional: result = compute(input)
// Sanguis:    emit compute(input) to result

The arrow to is the only operator that moves data. Everything else is either a hold declaration, a condition, or arithmetic inside an emit.

Programs are organisms, not functions

TraditionalSanguis
Files of functionsOrgans of an organism
main() starts, runs, exitsRoute ticks forever
You manage stateLayers manage state for you
You serialize/deserializeMemory layer IS persistence
Solve problemsGrow organisms

No garbage collection, no manual memory management

The layers manage memory. live values are overwritten every tick (no need to free them). learned values persist for the session. memory values are written to disk as .e3 files. aware values are derived — they're recomputed every breath from the other three. You never call malloc. You never call free. You never collect garbage. The route handles it.

The organism has a body

In traditional languages, I/O is a library call (printf, println, write). In Sanguis, the body is an organ:

emit "hello" to screen       // terminal output (like printf)
emit 42 to world              // exit code (like return 42)
emit write(data) to disk path // file I/O (like fwrite)

These aren't function calls. They're organs of the body, wired into the route like any other circuit.
