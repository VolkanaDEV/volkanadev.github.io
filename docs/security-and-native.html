Security — The Fuse

Five crypto layers, a one-shot kill switch, and what it all protects

VANA has layered cryptographic protection baked into the organism. Five layers, all stacked:

The 5 crypto layers

LayerAlgorithmPurpose
1SHA-256Fingerprint every core file at boot
2HMAC-SHA3-256Sign the integrity manifest (NIST SHA-3)
3AES-256-GCMInner encryption (hardware-accelerated)
4ChaCha20-Poly1305Outer encryption (wraps the AES ciphertext)
5Argon2idKey derivation from identity seed (64MB)

The .vana file is encrypted with both ciphers. ChaCha20 encrypts the AES ciphertext, so the AES output is never exposed in the clear. An attacker must break ChaCha20 then AES to read the genome.

The Fuse (kill switch)

How it works:

1. At install, VANA generates an identity seed (32 random bytes + 16-byte salt)
2. All keys are derived from the seed via Argon2id (64MB memory cost, 3 passes, 2 lanes)
3. Every core file is fingerprinted with SHA-256
4. The manifest (all fingerprints) is signed with HMAC-SHA3-256
5. The manifest is encrypted with both AES-256-GCM and ChaCha20-Poly1305, then saved to disk
6. At every boot, the system verifies its own integrity

If any file is modified or missing → the fuse blows:

The identity seed is zeroed in memory.
The on-disk identity file is overwritten with random bytes, then deleted.
The on-disk manifest is overwritten with random bytes, then deleted.
The copy stays intact — files are NOT removed, nothing is deleted except the key.
The copy is banned and on their own — it can't decrypt, can't verify, can't wink at home.
Any .vana files encrypted with that identity are permanently unreadable.

The fuse is one-shot. There is no recovery. A fused copy is scrap.

Remote verification ("wink at home")

Stubbed for now. The local integrity check is the sole authority. When the master server is built, the system will contact it with a derived public key. The server checks its key list and returns:

Valid: the installation is recognized
Invalid: the installation is not recognized (fuse blows)
Unreachable: fall back to local check

Debug output

The IdentitySeed struct has a manual Debug implementation that redacts the seed and salt:

IdentitySeed { seed: "[REDACTED]", salt: "[REDACTED]", fused: false }

Secrets never leak through debug output.

Native Compilation

.sang to .exe — no external compiler, no assembler, no linker

Pipeline

.sang  →  [parse]  →  Genome  →  [native codegen]  →  .exe (Windows PE)

The .exe is fully standalone — no Rust runtime, no dependencies.

What works natively

SyntaxNative output
emit N to worldExitProcess(N)
emit "text" to screenWriteFile(stdout, "text\n")
Multiple screen emitsN WriteFile calls in sequence
No emitExitProcess(0)

Usage

sanguis native hello.sang
# Produces hello.exe

sanguis native hello.sang -o custom.exe
# Produces custom.exe

V3 native pipeline (multi-stage)

.v3.sang  →  v3_compiler.sang  →  .vana genome  →  v3_native.sang  →  .exe
              (V2 .sang, runs on       (pipe-delimited     (V2 .sang, runs on
               sanguis.exe)             text format)        sanguis.exe)

When all V3 features compile natively, the compiler itself can be compiled to .exe, replacing sanguis.exe. That's 100% Sanguis. Zero Rust.

File Formats

Every file type in the Sanguis ecosystem

.sang — Sanguis source (the genome)

Human-written text file. Contains circuit/body declarations, holds, when blocks, but conditions, emit statements, and wires. Comments with //.

.sirc — SIRC (Sanguis Intermediate Representation Code)

Compiled netlist. Text-based. The VANA engine reads this to boot an organism.

# SIRC v2
# Generated from: hello.sang
# Circuits: 1  Holds: 1  Routes: 1  Wires: 0  Neurons: 1

CIRCUIT Hello
  HOLD greeting memory text "Hello, world!"
  ROUTE start
    EMIT greeting -> screen

.e3 — Episodic memory (the soul)

Binary format. One file per episode. See the Engine & Soul page for the format.

.vana — V3 genome format

Pipe-delimited text format used in the V3 native compilation pipeline:

VANA|v1
HOLD|name|type|default
EMIT|value|target|
WHEN|name
BUT|condition

.exe — Native Windows PE

Standalone executable. No Rust runtime. No dependencies. Generated by sanguis native.
